# 🚀 DevOps Learning Journey – Day 2

## 📚 Topic:

Linux File Editing and Permissions

## 🧠 What I Learned:

Today I learned how to edit files in Linux using the nano editor. I also learned how to read file content using the cat command.
I understood Linux file permissions and how access is controlled using read, write, and execute permissions.

## 💻 Commands I Practiced:

* nano     → Edit file
* cat      → View file content
* ls -l    → Check permissions
* chmod    → Change permissions

## 🧪 Practice Work:

* Edited a file using nano
* Wrote text inside the file
* Viewed file content using cat
* Checked file permissions
* Changed permissions using chmod

## 💡 Key Takeaways:

* File permissions are very important in Linux
* Only authorized users should get access
* nano is a simple editor for beginners
